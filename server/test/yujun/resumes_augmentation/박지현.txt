1) 결제/정산 도메인 API 서버 개발 및 운영
- 기술: Spring Boot, JPA, MySQL, Redis, AWS EC2/RDS/S3, Jenkins
- 역할:
  - 카드/계좌 결제, 환불, 부분취소, 정산 스케줄 등 주요 도메인 API 설계 및 구현
  - 외부 PG사 연동 모듈 개발 및 결제 결과 콜백 처리 로직 설계
  - Redis를 활용한 간단한 idempotency 처리(중복 결제 방지 토큰) 적용
  - 운영/CS 팀과 협업해 정산 리포트 쿼리 및 배치 작업 구현
- 성과:
  - 정산 배치 처리 시간을 90분 → 35분으로 단축 (쿼리 튜닝 및 배치 로직 리팩터링)
  - 일 평균 결제 요청 2배 증가 구간에서 별도의 리소스 증설 없이 장애 없이 운영
  - 결제 실패 로그/사유를 구조화하여 저장한 뒤, 반복적으로 발생하던 특정 케이스 실패율 4.3% → 1.2%로 감소

2) 관리자 포털 백엔드 및 권한 시스템 개편
- 기술: Spring Boot, Spring Security, MySQL, Redis
- 역할:
  - 운영팀/마케터용 관리자 포털 API 개발 (쿠폰 발급, 프로모션 설정, 정산 리포트 조회 등)
  - 기존 단순 "관리자 여부"만 체크하던 구조에서, 역할/권한 단위로 세분화된 RBAC 구조 설계 및 적용
  - 프론트 개발자와 함께 화면 단위 권한 맵을 정의하고 API 단 권한 체크 필터 구현
- 성과:
  - 잘못된 계정 권한으로 인한 민감 데이터 조회 사고 0건(권한 개편 이후 1년간)
  - 내부 운영 툴 UI 변경 시 서버 권한 로직 변경 없이 대응 가능하도록 구조화

3) 모니터링/장애 대응 체계 보완
- 기술: AWS CloudWatch, Prometheus, Grafana, Slack Webhook
- 역할:
  - 주요 비즈니스 지표(결제 성공률, 환불 비율, 재시도율)에 대한 메트릭 수집 및 대시보드 구성
  - 에러 레벨 및 빈도/비율 기준으로 알람 룰 설계 후 Slack 채널 알림 연동
  - 장애 발생 시 원인/대응/재발 방지 액션 아이템을 정리한 "Incident Postmortem" 템플릿 작성
- 성과:

  - 야간 시간대 장애 탐지까지 걸리던 평균 시간(TTD)을 약 40% 단축(내부 회고 기준)
