브로드링크(BroadLink) / 백엔드 개발자
2021.07 ~ 현재 (3년 5개월, 정규직, 성남)

1) 커머스 플랫폼 API 서버 개발 및 운영
- 기술: Node.js, NestJS, PostgreSQL, Redis, AWS (EC2, RDS, SQS, CloudWatch)
- 역할:
  - 상품 리스트/상세, 장바구니, 주문/결제, 배송 조회 등 주요 도메인 API 설계 및 개발
  - 클라이언트(웹/앱)와 논의하여 응답 스키마와 페이징 구조, 에러 코드 규격 정의
  - Redis 캐시를 이용한 상품 목록/배너 조회 API 응답 속도 개선
  - 주문 처리 배치 및 외부 PG사 연동 로직 리팩터링
- 성과:
  - 메인 상품 리스트 API P95 응답 시간 480ms → 210ms로 개선
  - 캐시 도입 후 DB 조회량 약 55% 감소, 피크 타임 CPU 사용률 30% 이상 감소
  - 공통 에러 응답 포맷(에러 코드/메시지/트레이싱 ID) 도입 후, 클라이언트 팀의 장애 원인 파악 시간 단축(내부 회고 기준 평균 20~30분 단축)

2) 관리자 콘솔 백엔드 및 권한 시스템 개편
- 기술: NestJS, TypeORM, PostgreSQL, AWS Cognito
- 역할:
  - 마케터/CS/운영팀용 관리자 콘솔 API 개발 (프로모션 설정, 쿠폰 발급, 배너 노출 제어 등)
  - RBAC(Role-Based Access Control) 기반 권한 모델 설계 및 마이그레이션
  - 운영팀과 협업하여 화면 단위의 권한 정책을 정의하고 API 단 권한 체크 로직 구현
- 성과:
  - 기존 SQL 스크립트 + 수동 작업으로 처리하던 프로모션 세팅을 관리자 콘솔에서 처리 가능하게 전환, 관련 작업 평균 처리 시간 1일 → 1시간 내로 단축
  - 권한 개편 이후, 잘못된 권한으로 인한 운영 사고 0건(6개월 기준)

3) 로그/모니터링 및 장애 대응 프로세스 개선
- 기술: AWS CloudWatch, Sentry, Slack Webhook, Shell Script
- 역할:
  - 주요 API에 대해 비즈니스 지표(주문 성공률, 결제 실패율, 재시도 비율) 로그 구조 정리
  - 에러 레벨/빈도에 따른 알람 규칙 설계 및 Slack 채널 알림 연동
  - 장애 발생 시 재발 방지 RCA 문서 템플릿 정리, 공용 Confluence 공간에 공유
- 성과:
  - 배포 후 1시간 이내 발견되는 장애 비율 증가(사전 탐지)로, 고객 문의 발생 전에 조치한 사례 다수
  - 이전에는 QA/CS 제보로만 인지되던 일부 결제 실패 케이스를 선제적으로 탐지하여, 해당 구간 오류율 3.1% → 0.6%로 감소
